## In-progress work: dropping Jekyll for a self-contained static site

**Branch:** `static-site` (created off `jekyll-site-claude-edits`).

**Goal:** Convert this site from a Jekyll-built site (Liquid templating in `index.md` + `_includes/*.html` + `_layouts/default.html`) to a plain self-contained static site that can be hosted anywhere with zero build step — while keeping the photo gallery self-updating (not a hand-maintained hardcoded list).

Full plan written at `/Users/nik/.claude/plans/velvet-humming-hamster.md` — read that for full rationale/background.

### What's been done (uncommitted, staged via `git add -A` on the `static-site` branch)

1. **No Jekyll install/build was needed.** All Liquid in this site was trivial enough to resolve by hand directly from the source `_includes/*.html` files (confirmed: `{{ page.title }}` → literal `"Home"`, `relative_url` is a no-op since `baseurl: ""`, no `{% seo %}` tag anywhere). The committed `_site/` build output was stale (59 vs 61 gallery images) and was not used as a source.
2. **Gallery made self-sufficient without Jekyll:**
   - `assets/images/gallery/manifest.json` — JSON array of the 61 filenames in that folder, generated by:
   - `scripts/update-gallery-manifest.sh` — dependency-free shell script that re-lists `assets/images/gallery/*.jpg` and rewrites `manifest.json`. Run this after adding/removing a gallery photo, then commit.
   - `index.html`'s gallery section is now just `<div id="lightgallery"></div>`; an inline `<script>` fetches `manifest.json` at page load, builds the `<a class="gallery-item"><img></a>` elements, then initializes `lightGallery(...)` on the populated container (sequencing matters — init must run after the manifest-driven DOM build).
3. **Flattened `index.html`** at repo root — hand-assembled from `_layouts/default.html` + all the includes (`intro`, `claim`, `hero`, `about`, `gallery`, `contact`, `artverwandt`) in `index.md`'s original order, each wrapped in an HTML comment naming the former section. All other JS (Vivus signature SVG draw-in, background slideshow, menu overlay, scroll-to-top) carried over unchanged from `_layouts/default.html`.
4. **Static assets carried over as-is:** `assets/css/index.css`, `assets/images/` (incl. `gallery/`), `CNAME`, `favicon.ico`, `ads.txt`.
5. **Hand-authored SEO files** (the old `_site/robots.txt` / `sitemap.xml` baked in `http://0.0.0.0:4000` from a local dev build, so they were rewritten rather than copied): `robots.txt` and `sitemap.xml` now point at `https://altmannfotografie.at/`. Dropped `feed.xml` (jekyll-feed output for a blog with no posts — not useful).
6. **Added `.nojekyll`** at root (tells GitHub Pages not to run its own Jekyll build over the flattened output).
7. **Removed the entire Jekyll/Ruby toolchain:** `_config.yml`, `_config.bu.yml`, `_layouts/`, `_includes/` (incl. dead, unreferenced `header.html`/`footer.html`), `Gemfile`, `Gemfile.lock`, `vendor/`, `.bundle/`, `.sass-cache/`, `Dockerfile`, `build.sh`, `run.sh`, `jekyllCommands.txt`, `.travis.yml`, `index.md`, `_site/`, `.dockerignore`. This also fixed the pre-existing repo-hygiene issue where `vendor/`/`.bundle/`/`_site/` were bloating `.git` to ~114MB.
8. **Added `.gitignore`** (`.DS_Store`) and removed stray tracked `.DS_Store` files.
9. **Rewrote `README.md`** — the old one was the unmodified upstream "rampatra/photography" Jekyll theme template, not actually describing this project. New README explains it's a plain static site, how to run it locally (`python3 -m http.server`), and the gallery-manifest update workflow.
10. Ran `git add -A` to stage the new tree (git shows some `vendor/.../foo -> .nojekyll` style entries — these are just rename-detection noise from the toolchain deletion, harmless).
